How this tool works

A permanent, free file host built on a Hugging Face dataset repo, served through a serverless FastAPI backend on Vercel. Here's exactly what happens behind each button.

1. The building blocks

There are three pieces: the frontend (this website, plain HTML/JS), the backend (a Python FastAPI app running as a Vercel serverless function), and storage (a public Hugging Face dataset repo, which gives ~unlimited free storage and a permanent, directly-linkable URL per file).

2. Uploading a file

Drop a file (or several, or paste one) and the browser sends it straight to /api/upload as multipart/form-data. No file ever touches disk on the server — it's read into memory and streamed to Hugging Face.

  • Any file extension is accepted — the server never inspects file type, only reuses the extension for the generated ID.
  • Max size is ~4.4 MB per file, kept just under Vercel's hard 4.5 MB request-body limit.
  • A custom ID is only allowed when uploading exactly one file, and is sanitized to [a-zA-Z0-9._-] to block path traversal.

3. Permanent link vs. expiring link

The link you get back depends on whether you picked an expiry. This matters because it changes how the link is served when someone opens it later.

  • Permanent links are handled entirely by Vercel's static redirect rules — the Python function never even runs, so they're as fast as a link can be.
  • Expiring links must go through the backend on every open, since something has to check "has this passed its expiry time yet?" before deciding where to send you.
  • Nothing is ever deleted. An expired link just stops redirecting — the file stays in the dataset.

4. Uploading from a URL

Instead of uploading from your device, you can hand the tool a link and have the server fetch and re-host it — useful for re-hosting an image or file that's only temporarily available elsewhere.

Same 4.4 MB size limit applies to the fetched file, and it supports the same custom ID and expiry options.

5. Batch links

Upload more than one file at once and, alongside each file's own link, you also get a single shareable /batch/<id> link that opens a page listing every file from that batch. If you picked an expiry when uploading, that same expiry applies to the batch page itself — once it passes, the batch link stops working too, even though the individual files' expiry is still checked separately each time you open them.

6. Limits & behavior at a glance

Max file size ~4.4 MB per file (Vercel request-body limit)
File types Any extension, unrestricted
Multiple files Supported; each gets its own link + one shared batch link
Custom ID Single-file uploads only, sanitized to letters/numbers/dot/dash/underscore
Expiry Optional: 1h / 1d / 7d / 30d / never. Disables the link only — the file itself is never deleted
Deletion Not supported by design
Authentication None — the tool is intentionally open
Storage Public Hugging Face dataset repo
